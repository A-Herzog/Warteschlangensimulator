Java-Plugins-Verzeichnis Dialog

In den Skript-Stationen kann von dem internen Java-Code aus Java-Code in externen Klassendateien aufgerufen werden. Dafür stehen zwei Methoden zur Verfügung:

Plugin-Schnittstelle

Externe Klassen können Methoden mit folgender Signatur implementieren:

public Object methodName(final scripting.java.RuntimeInterface runtime, final scripting.java.SystemInterface system, final Object data)

Auf diese Methoden kann im internen Java-Code über folgende Methode zugegriffen werden:

sim.getSystem().runPlugin("className","methodName",data)

Wird in dem Dialog ein Verzeichnis gewählt, so werden in der Listenansicht automatisch die jeweils gefundenen Klassendateien und die darin enthaltenen, passenden Methoden aufgelistet. Es werden dabei nur die Klassen aufgelistet, die Methoden gemäß der oben beschriebenen Signatur enthalten.

Import externer Klassen

Externe Klassen, die sich in den Unterverzeichnissen des angegebenen Verzeichnisses befinden, können direkt über import-Anweisungen verfügbar gemacht werden. Zu beachten ist dabei, dass sich die Klassen nicht in dem angegebenen Plugin-Verzeichnis selbst befinden dürfen, sondern in dessen Unterverzeichnissen und dass in den Klassen ein passender package-Bezeichner eingetragen sein muss.

Der Import erfolgt automatisch durch den Simulator. Es müssen keine manuellen import-Konfigurationen vorgenommen werden. Es muss nur das Basisverzeichnis, in dem sich die externen Klassen befinden, angegeben werden.

Allgemeine Hinweise

  • Eine allgemeine Dokumentation der verfügbaren Java-Befehle steht auf der Hilfe-Seite Dokumentation der Java-Befehle zur Verfügung.
  • Wurde eine Klassendatei erstmalig durch den Simulator geladen bzw. verwendet, so kann diese bis zum Beenden des Simulators gesperrt bleiben. Das bedeutet, dass insbesondere nicht sichergestellt ist, dass class-Dateien im Plugins-Verzeichnis zur Laufzeit des Simulators ausgetauscht werden können.
