Stapelverarbeitung

Die Stapelverarbeitungsfunktion ermöglicht die Anwendung von zwei Arten von Funktionen auf jeweils alle passenden Dateien in dem ausgewählten Verzeichnis:

Simulation

In dieser Betriebsart werden alle Modelle und Parameterreihenkonfigurationen, die sich in dem angegebenen Verzeichnis befinden, verarbeitet. Heißt eine Datei Modell.xml, so werden die Ergebnisse dazu in Results-Modell.zip gespeichert. Auf diese Weise kann der Simulator auch prüfen, zu welchen Eingabedateien bereits Simulationsergebnisse vorliegen. Wird die Verarbeitung eines Verzeichnisses unterbrochen, so werden bei einer Wiederaufnahme der Verarbeitung nur die Modelle bzw. Parameterreihenkonfigurationen bearbeitet, für die noch keine Ausgabedateien vorliegen.

Diese Funktion entspricht dem Kommandozeilenbefehl Verzeichnis.

Ergebnisse filtern

In dieser Betriebsart wird ein angegebenes Filterskript auf alle Statistikdateien, die in einem Verzeichnis vorliegen, angewandt. Die Ergebnisse werden gesammelt in eine gemeinsame, vom Nutzer angegebene Ausgabedatei geschrieben.

Diese Funktion entspricht dem Kommandozeilenbefehl VerzeichnisFilter.
