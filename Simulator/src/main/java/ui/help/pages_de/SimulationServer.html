Netzwerkserverdienste

Siehe auch Abschnitt Netzwerksimulation im Lehrbuch.

Der Warteschlangensimulator bietet zwei Netzwerkserverdienste an, die über diesen Dialog konfiguriert und gestartet und gestoppt werden können. Die beiden Dienste stehen nicht in Verbindung miteinander und können unabhängig voneinander verwendet werden.

Simulationsnetzwerkserver

Der Simulationsnetzwerkserver-Dienst ermöglichst es, die aktuelle Warteschlangensimulator Instanz als Rechenserver für entfernte Installationen zu verwenden. Wenn der Serverdienst gestartet wurde, kann die Programmdialogseite geschlossen werden; der Server läuft bis zum Programmende im Hintergrund weiter.

Soll eine Warteschlangensimulator Installation auf diesen Server zugreifen, so muss dies über den Einstellungen-Dialog konfiguriert werden.

Webserver

Der Webserver ermöglicht es dem Warteschlangensimulator, über ein Webinterface Arbeitsaufträge in Form von Modell-xml-Dateien und Parameterreihen-xml-Dateien zu übermitteln. Die Anfragen werden in Eingangsreihenfolge bearbeitet und die Ergebnisse als Statistik-xml-Dateien zum Download angeboten.

Fernsteuerungs-Server

Der Fernsteuerungs-Server ermöglicht es, den Status einer Animation über das http-Protokoll in Form von json-Objekten abzufragen und die Animation ebenfalls über diese Schnittstelle zu steuern.

MQTT-Klient

Der Warteschlangensimulator beinhaltet einen MQTT-Klienten, der sich bei einem MQTT-Broker anmelden kann. Der MQTT-Klient abonniert bei dem Broker ein Thema, über das dem Simulator Arbeitsaufträge übermittelt werden können. Zusammen mit dem Arbeitsauftrag muss ein Antwortthema angegeben werden (d.h. es wird MQTT v5 benötigt), über das der Simulator die Ergebnisse zurückliefert.

Optional kann noch ein zweites Thema angegeben werden über das der Simulator in regelmäßigen Abständen Informationen zur aktuellen Systemauslastung an den Server übermittelt. Während für das Arbeitsaufträge-Thema ein Name angegeben werden muss, kann das Feld für das Statusthema leer gelassen werden. Es erfolgen dann keine Ausgabe zur Auslastung.

DDE-Server

Der DDE-Server ermöglicht es, den Status einer Animation über das Dynamic Data Exchange Protokoll abzufragen und die Animation ebenfalls über diese Schnittstelle zu steuern. Der Simulator registriert sich dabei als Service "QS" und bietet die Themen "Status" und "Calc" an. "Status" besitzt die vier Datenfelder "Version", "Model", "Changed" und "Mode". An das Thema "Calc" können als anzufragende Datenfelder Rechenbefehle, die dann im Kontext der Animation berechnet werden, übergeben werden. Des Weiteren stehen die beiden DDE-Befehle "step" und "quit" zur Steuerung einer laufenden Animation zur Verfügung.

Socket-basierter Rechenserver

Der Socket-basierter Rechenserver ermöglicht, es dem Warteschlangensimulator über Socket-Verbindungen Arbeitsaufträge in Form von Modell-xml-Dateien und Parameterreihen-xml-Dateien zu übermitteln. Die Anfragen werden in Eingangsreihenfolge bearbeitet und die Ergebnisse als Statistik-xml-Dateien zum Download angeboten.
