Einstellungen-Dialog

Der Einstellungen-Dialog kann über den Menüpunkt Datei|Einstellungen aufgerufen werden.

In dem Einstellungen-Dialog kann die Basiskonfiguration des Simulators vorgenommen werden. Der Dialog besteht aus folgenden Registerreitern:

  Benutzeroberfläche

Anzeigesprache

In dem Feld kann ausgewählt werden, ob die Programmoberfläche auf Deutsch oder auf Englisch angezeigt werden soll.

(Ist als Systemsprache Deutsch gewählt, so wird als Vorgabewert "Deutsch" eingestellt, sonst "Englisch".)

Rechtschreibprüfung

Im Lieferumfang des Warteschlangensimulators befinden sich Wörterbücher für Deutsch und Englisch. Auf dieser Basis können die Eingaben in mehrzeilige Textfelder (Modellbeschreibungen, Textelemente, usw.) geprüft werden. In dem Konfigurationsdialog für die Rechtschreibprüfung können einzelne Wörterbücher gezielt ein- oder ausgeschaltet werden und es kann die Liste der vom Nutzer hinzugefügten Wörter bearbeitet werden.

Die Rechtschreibprüfung kann Wörterbücher im xpi- und im oxt-Format verwenden. Bei den xpi-Dateien handelt es sich um Firefox-Extensions, bei den oxt-Dateien um OpenOffice/LibreOffice-Erweiterungen. Weitere Wörterbücher können daher von folgenden Adressen bezogen werden:

  • Mozilla Language Tools
  • OpenOffice Extensions

Eine Integration der zusätzlichen Wörterbücher ist direkt über den Rechtschreibprüfung-Dialog möglich.

Thema

In Java stehen (abhängig vom verwendeten Betriebssystem) verschiedene Themen zur Gestaltung der grafischen Programmoberfläche zur Verfügung. Über dieses Feld kann das zu verwendende Thema gewählt werden.

(Als Standard ist "Systemvorgabe", d.h. die Anzeige des Programms in der Form, wie es für native Programme vorgesehen ist, gewählt.)

Menü in Titelzeile

In den Flat-Themen kann, wenn der Simulator unter Windows 10 läuft, das Menü direkt in die Titelzeile integriert werden. Ist diese Option aktiviert, so erfolgt diese Integration und es steht entsprechend eine zusätzliche Zeile an Platz für das Modell im Programmfenster zur Verfügung. Wird der Simulator unter einem anderen Betriebssystem als Windows 10 ausgeführt oder ist keines der Flat-Themen gewählt, so ist diese Option ohne Bedeutung.

(Diese Option ist als Vorgabewert aktiviert.)

Schriftgröße für Programmoberfläche

In diesem Auswahlfeld kann eingestellt werden, ob die Schriftgrößen (und damit auch die Fenstergrößen) in der grafischen Oberfläche des Programms für eine bessere Lesbarkeit skaliert werden sollen. Veränderungen der Schriftgröße werden erst nach einem Neustart des Programms wirksam.

(Vorgabewert ist für diese Option: "Normal".)

Hohe Kontraste verwenden

Wird diese Option aktiviert, so werden die Elemente auf der Zeichenfläche mit einem durchgängig hohen Kontrast dargestellt. Dafür werden individuelle Farben für die Stationen sowie Farbverläufe deaktiviert.

(Diese Option ist als Vorgabewert deaktiviert.)

Schriftgröße in Skript-Editorfeldern

In diesem Auswahlfeld kann die Schriftgröße (in Punkten), die in den Eingabefeldern für Javascript- und Java-Code verwendet werden soll, angegeben werden. Diese Einstellung ist unabhängig von der allgemeinen Schriftgröße für die Programmoberfläche.

(Die Standardschriftgröße für Skript-Editorfelder beträgt 13 Punkte.)

Modell automatisch speichern

In diesem Auswahlfeld kann eingestellt werden, ob das Modell nie, vor dem Start von Simulationen/Animationen oder nach jeder Änderung am Modell automatisch gespeichert werden soll.

(Als Vorgabewert ist die automatische Speicherung deaktiviert.)

Zuletzt verwendete Modelle zum Laden anbieten

Ist diese Option aktiviert, so werden im Dateimenü im Untermenü "Zuletzt verwendet" Menü die als letztes geladen oder gespeicherten Modell zum erneuten Laden angeboten.

(Als Vorgabeeinstellung werden die zuletzt verwendeten Modelle im Dateimenü zum Laden angeboten.)

Zuletzt bearbeitetes Modell beim nächsten Programmstart wiederherstellen

Ist diese Option aktiviert, so wird das zuletzt im Editor befindliche Modell beim Beenden automatisch intern gespeichert und beim nächsten Programmstart wieder in den Editor geladen. Ein explizites Speichern des Modells als Datei ist dafür nicht erforderlich.

(Als Vorgabewert ist die automatische Wiederherstellung deaktiviert.)

Fenstergröße beim Programmstart

In diesem Auswahlfeld kann eingestellt werden, wie das Programmfenster beim Start des Simulators aussehen soll. Neben "Vorgabefenstergröße" und "Vollbildmodus" steht über die Funktion "Letzte Fenstergröße wiederherstellen" insbesondere die Option zur Verfügung, die Größe des Fensters beim Programmende zu speichern und diese beim nächsten Start wiederherzustellen.

(Vorgabewert für diese Option: "Vorgabefenstergröße".)

Element-Vorlagen beim Programmstart

Über dieses Auswahlfeld kann eingestellt werden, ob die Element-Vorlagen-Leiste am linken Rand des Modell-Editor-Fensters beim Programmstart angezeigt oder ausgeblendet werden soll. Die Leiste kann jederzeit über die Schaltfläche "Elemente" in der linken vertikalen Symbolleiste ein- oder ausgeblendet werden.

(Vorgabewert für diese Option: "Ausblenden".)

Modell beim Starten öffnen

Beim Programmstart kann statt einer leeren Zeichenfläche auch direkt ein Beispielmodell geladen werden. In diesem Auswahlfeld kann eingestellt werden, ob ein solches Beispielmodell geladen werden soll.

(Vorgabewert für diese Option: "Leeres Modell".)

Hinweise auf Zeichenfläche anzeigen

Auf der Zeichenfläche können, wenn diese leer ist, optional Hinweise zur Verwendung der vertikalen Symbolleiste und zum Abruf von Hilfe-Informationen angezeigt werden.

(Als Vorgabe werden Hinweise auf der Zeichenfläche nur direkt nach dem Programmstart angezeigt.)

Halbtransparente Informationen auf der Zeichenfläche anzeigen

Wird das Mausrad auf der Zeichenfläche verwendet oder werden Elemente auf der Zeichenfläche mit der Maus verschoben, so wird eine halbtransparente Information eingeblendet, mit welchen Umschalttasten welche weiteren Aktionen ausgelöst werden können.

(Diese Option ist als Vorgabewert aktiviert.)

Benachrichtigung zu Simulationsende anzeigen

Über dieses Auswahlfeld kann eingestellt werden, ob nach dem Ende einer Simulationsende, einer Parameterreihen-Simulation oder einer Optimierung eine Systembenachrichtigung angezeigt werden soll.

(Als Vorgabe werden Benachrichtigung nur bei langen Simulationsläufen ausgegeben.)

Hinweise in Dialogen anzeigen

Über diese Schaltfläche kann ein Dialog geöffnet werden in dem konfiguriert werden kann in welchen Dialogen zusätzliche Hinweismeldungen im oberen Fensterbereich eingeblendet werden sollen und ob beim Programmstart eine Begrüßungsseite angezeigt werden soll.

(Als Vorgabe sind alle Hinweismeldungen aktiviert, die Begrüßungsseite jedoch deaktiviert.)

  Leistung

Hintergrundverarbeitung

Über dieses Auswahlfeld kann eingestellt werden, wie viel (Vor-)Verarbreitung der Simulator bereits im Hintergrund für ein Modell durchführen soll. Je mehr Schritte bereits während der Arbeit im Modell-Editor im Hintergrund durchgeführt werden, desto weniger Zeit benötigt später die Simulation aber auch desto mehr Computerleistung ist notwendig.

(Der Vorgabewert für diese Option lautet "Modell im Hintergrund simulieren wenn sinnvoll".)

Animationen in der Programmoberfläche aktivieren

Ist diese Option aktiviert, so werden Ein- und Ausblende-Animationen beim Aktivieren und Deaktivieren der Elementenvorlagenleiste und des Modellnavigators angezeigt.

(Per Vorgabe sind die Animationen aktiviert.)

Mehrere CPU-Kerne für Simulation nutzen

Ist diese Option aktiviert, so wird die Simulation über alle verfügbaren CPU-Kerne verteilt. Dies führt dazu, dass die benötigte Rechenzeit deutlich sinkt, gleichzeitig aber der Arbeitsspeicherbedarf steigt.

(Diese Option ist als Vorgabewert aktiviert.)

Wiederholte Simulationen wenn sinnvoll aufteilen

Ist diese Option aktiviert und ist zusätzlich die Mehrkernsimulation als solches aktiviert, so wird die eingestellte Anzahl an Wiederholungen evtl. durch den Simulator erhöht. Soll ein Modell mehrfach simuliert werden, so werden die einzelnen Wiederholungen auf die verfügbaren CPU-Kerne verteilt. Ist die eingestellte Anzahl an Wiederholungen kleiner als die Anzahl an verfügbaren CPU-Kernen, so würden einige Kerne überhaupt nicht verwendet. Um die verfügbare CPU-Leistung optimal auszunutzen, wird, wenn diese Option aktiviert ist, die Anzahl an Wiederholungen so lange verdoppelt und gleichzeitig die Anzahl an Kundenankünften pro Wiederholung so lange halbiert, bis alle CPU-Kerne genutzt werden. Dies führt dazu, dass sich bessere Konfidenzintervalle ergeben, als eigentlich durch die Benutzereinstellung zur Anzahl an Wiederholungen der Simulation zu erwarten gewesen wäre. Daher ist diese Option als Vorgabe deaktiviert.

(Diese Option ist als Vorgabewert deaktiviert.)

Mehrere CPU-Kerne für Animation nutzen

Ist diese Option aktiviert, so wird die Animation über mehrere verfügbaren CPU-Kerne verteilt. Es werden dabei Simulation und Anzeige in separaten Threads ausgeführt.

(Diese Option ist als Vorgabewert aktiviert.)

Hohe Priorität für Simulation

Ist diese Option aktiviert, so wird die Simulation mit einer höheren Priorität in Bezug auf die verfügbare Rechenleistung durchgeführt. Dies kann die Simulation geringfügig beschleunigen, aber auch dazu führen, dass andere parallel ausgeführte Anwendungen während der Simulation verzögert reagieren.

(Diese Option ist als Vorgabewert deaktiviert.)

NUMA-Unterstützung für Simulation

Ist diese Option aktiviert, so verwendet der Simulator für jeden Simulationsthread ein eigenes, unabhängiges Datenmodell. Dadurch erhöht sich der Speicherverbrauch. Allerdings kann so auf Systemen, bei denen nicht alle Speicherbereiche von allen CPU-Kernen mit derselben Geschwindigkeit angesprochen werden können (NUMA-Systeme), die Daten so im Speicher verteilt werden, dass sie möglichst effizient angesprochen werden können.

(Diese Option ist als Vorgabewert deaktiviert.)

Bevorzugte Javascript-Umgebung

Über dieses Auswahlfeld kann eingestellt werden, welche Javascript-Umgebung verwendet werden soll, um Javascript-Code im Warteschlangensimulator auszuführen. Zur Auswahl steht die in die Java-Laufzeitumgebung eingebaute Javascript-Umgebung (schneller, aber evtl. in zukünftigen Java-Versionen nicht mehr vorhanden), die Mozilla Rhino Umgebung (langsamer, immer verfügbar) sowie die GraalJS-Umgebung (nur verfügbar, wenn unter GraalVM ausgeführt). Ist "Automatisch wählen" gewählt, so wird die jeweils schnellste verfügbare Umgebung verwendet.

(Als Vorgabewert ist "Automatisch wählen" gewählt.)

Der Name der jeweils verwendeten Umgebung kann über die globale Javascript-Variable JS_ENGINE_NAME abgefragt werden.

Simulation bei Skriptfehler abbrechen

Soll die Simulation oder Animation abgebrochen werden, wenn die Ausführung eines Skriptes an einer Station zu einem Fehler führt?

(Diese Option ist als Vorgabewert aktiviert.)

Maximale Laufzeit für Javascript-Code

Über dieses Feld kann eingestellt werden, wie lange nutzerdefinierer Javascript-Code zur Filterung von Statistikergebnissen und während einer Simulation jeweils maximal laufen darf, bevor er, um Blockierungen des Systems zu vermeiden, abgebrochen wird. Die Einstellung bezieht sich nur auf Javascript-Code; Java-Code an denselben Stellen kann nicht zeitgesteuert abgebrochen werden.

(Der Vorgabewert für diese Eigenschaft sind 2 Sekunden.)

Simulationen auf entferntem Server durchführen

Ist diese Option aktiviert und sind passende Zugangsdaten zu einem Simulationsserver eingetragen, so versucht der Simulator jede Simulation auf diesem Server auszuführen. Ist der Server nicht erreichbar, so erfolgt die Simulation lokal. Die Animation wird ebenfalls stets lokal ausgeführt.

(Diese Option ist als Vorgabewert deaktiviert.)

  Animation

Einschwingphase während der Animation

Während der Einschwingphase erfolgt noch keine Erfassung von Statistikdaten. Wird das Modell nicht simuliert, sondern als Animation dargestellt, so zeigen evtl. enthaltene Statistik-Visualisierungselemente (Diagramme, Zahlwertdarstellungen, Balkenvisualisierungen) keine sinnvollen Werte an, wenn diese Statistikdaten (z.B. mittlere Anzahl an Kunden an einer Bedienstation) anzeigen sollen. Daher ist es sinnvoll, in diesem Fall die Einschwingphase zu überspringen oder vorab ohne Animationsanzeige zu simulieren.

(Vorgabewert für diese Option: Überspringen, wenn Statistikvisualisierungen verwendet werden.)

Lautzeitdaten zu den Stationen anzeigen

Ist diese Option aktiviert, so werden während der Animation unter den Stationen Daten zu der Anzahl an Kunden die die jeweilige Station bisher passiert haben und ggf. auch dazu, wie viele Kunden sich momentan in der Station aufhalten, angezeigt.

(Diese Option ist als Vorgabewert aktiviert.)

Logging-Daten im Einzelschrittmodus anzeigen

Wenn diese Option aktiv ist, werden, wenn diese Animation pausiert wurde und nun im Einzelschrittmodus ausgeführt wird, nach jedem Animationsschritt die Ereignisse, die bearbeitet wurden, in der Statuszeile des Programmfensters angezeigt.

(Diese Option ist als Vorgabewert aktiviert.)

Animation im Einzelschrittmodus starten

Ist diese Option aktiviert, so wird die Animation direkt nach dem Start auf Pause geschaltet, so dass sie im Einzelschrittmodus ausgeführt werden kann.

(Diese Option ist als Vorgabewert deaktiviert.)

Animation am Ende pausieren

Wird eine Animation eines Modells kontinuierlich (d.h. nicht im Einzelschrittmodus) ausgeführt, so wechselt der Simulator nach der Ausführung des letzten Ereignisses automatisch in die Statistikansicht. Wird diese Option aktiviert, so schaltet der Simulator hingegen zunächst in den Einzelschrittmodus, so dass die Modelldaten noch ein letztes Mal eingesehen werden können, bevor dann der nächste Schritt den Wechsel in die Statistikansicht veranlasst.

(Diese Option ist als Vorgabewert deaktiviert.)

Ressourcen und Transporter in der Animation anzeigen

Ist diese Option aktiviert, so werden während der Animation nicht nur die Kunden dargestellt, sondern auch die Nutzung von Ressourcen an den Stationen sowie die Transporter.

(Diese Option ist als Vorgabewert aktiviert.)

Animation verlangsamen um die Veränderung analoger Werte besser abzubilden

Wenn diese Option aktiviert ist, die Animation nicht im höchsten Geschwindigkeitsmodus läuft und das Modell zeitkontinuierliche Werte enthält, so wird die Animation verlangsamt, um die Veränderung dieser Werte exakter darzustellen. (Auf die Statistikerfassung in der Simulation hat dies keinen Einfluss.)

(Diese Option ist als Vorgabewert aktiviert.)

  Statistik

Quantile anzeigen

Ist diese Option aktiviert, so werden auf den Statistikseiten zusätzlich zu den üblichen Statistikkenngrößen auch die 10%-, 25%-, 50%-, 75%- und 90%-Quantile ausgegeben.

(Diese Option ist als Vorgabewert aktiviert.)

Erlang-C-Vergleich anzeigen

Ist diese Option aktiviert, so wird eine Statistikseite angeboten, auf der die Ergebnisse der Simulation denen eines von dem Simulationsmodell abgeleiteten Erlang-C-Modells gegenübergestellt werden.

(Diese Option ist als Vorgabewert aktiviert.)

Hinweis zu den Statistikwerten anzeigen

Ist diese Option aktiviert, so werden Hinweise in der Statistikansicht angezeigt, wenn einzelne Kenngrößen auffällige Werte (z.B. lange Wartezeiten für einzelne Kundentypen usw.) aufweisen. Auf diese Weise können Flaschenhälse oder auch Fehler im Modell schneller aufgespürt werden.

(Diese Option ist als Vorgabewert aktiviert.)

Statistikbaum immer sofort vollständig ausklappen

Im Normalfall werden im Statistikbaum aus Übersichtlichkeitsgründen einige eher selten genutzte Rubriken nach einer Simulation zunächst eingeklappt dargestellt. Ist diese Option aktiviert, so werden jedoch alle Rubriken immer sofort ausgeklappt.

(Diese Option ist als Vorgabewert deaktiviert.)

Nachkommastellen für Zahlenwerte

Über dieses Feld kann eingestellt werden, wie viele Nachkommastellen in der Statistikansicht in Texten und Tabellen für normale Zahlenwerte verwendet werden sollen. Mögliche Werte liegen zwischen 1 und 9. Bei einigen Statistikausgaben sind jedoch Nachkommastellenanzahlen größer als 1 fest vorgegeben, so dass hier stets mehr als eine Nachkommastelle ausgegeben wird. Ist der hier eingestellte Wert jedoch größer als diese Vorgabe, so wird dieser Wert berücksichtigt.

(Der Vorgabewert für diese Option lautet 1.)

Nachkommastellen für Prozentwerte

Über dieses Feld kann eingestellt werden, wie viele Nachkommastellen in der Statistikansicht in Texten und Tabellen für Prozentwerte verwendet werden sollen. Mögliche Werte liegen zwischen 1 und 9. Bei einigen Statistikausgaben sind jedoch Nachkommastellenanzahlen größer als 1 fest vorgegeben, so dass hier stets mehr als eine Nachkommastelle ausgegeben wird. Ist der hier eingestellte Wert jedoch größer als diese Vorgabe, so wird dieser Wert berücksichtigt.

(Der Vorgabewert für diese Option lautet 1.)

Anzeige von Quantilwerten

Normalerweise werden Quantile zu den Niveaus 10%, 25%, 50%, 75% und 90% ausgegeben. Durch eine Eingabe von durch Semikolon getrennten Wahrscheinlichkeiten bzw. Prozentwerten in diesem Feld können jedoch eigene Niveaus definiert werden.

(Im Vorgabefall ist dieses Feld leer, d.h. es werden Qantile zu den Niveaus 10%, 25%, 50%, 75% und 90% ausgegeben.)

Konfidenzniveaus für Batch-Means-Konfidenzintervalle

Wird die Aufzeichnung der Konfidenzdaten auf Basis der Batch-Means-Methode durchgeführt, so stehen nach der Simulation Konfidenzintervalle für Warte-, Transfer-, Bedien- und Verweilzeiten der Kunden zur Verfügung. Normalerweise werden Konfidenzintervalle zu den Konfidenzniveaus 90%, 95% und 99% ausgegeben. Durch eine Eingabe von durch Semikolon getrennten Wahrscheinlichkeiten bzw. Prozentwerten in diesem Feld können jedoch eigene Konfidenzniveaus, zu denen Konfidenzintervalle ausgegeben werden sollen, definiert werden.

(Im Vorgabefall ist dieses Feld leer, d.h. es werden Konfidenzintervalle zu den Niveaus 90%, 95% und 99% ausgegeben.)

Ergebnisse in externen Anwendungen öffnen

In dieser Rubrik kann eingestellt werden, welche externen Programme zur Übernahme der Daten in den Text- und Tabellen-Statistikansichten angeboten werden sollen.

(Als Vorgabe sind Word und Excel gewählt.)

Unvollständige Statistik

In den beiden Auswahlboxen in dieser Rubrik kann eingestellt werden, wie sich der Simulator verhalten soll, wenn eine Animation oder eine Simulation vorzeitig abgebrochen wird. Die bis zu diesem Zeitpunkt gesammelten Statistikdaten vermitteln evtl. noch keinen vollständigen Überblick über das Modell, können aber möglicherweise dennoch nützlich sein. Es kann eingestellt werden, ob die unvollständigen Daten verworfen werden sollen oder ob diese wie nach einem erfolgreichen Abschluss angezeigt werden sollen.

(Als Vorgabe werden unvollständige Statistikdaten nach dem Abbruch einer Animation verworfen und nach dem Abbruch einer Simulation wird der Nutzer gefragt, ob die Daten angezeigt werden sollen.)

  Dateiformate

Vorgabe-Autorenname für neue Modelle

Der in diesem Eingabefeld eingetragene Name wird als Autorenname beim Anlegen von neuen Modellen verwendet und wird bei der Durchführung von als Simulationsanwender erfasst. Der Autorenname kann für individuelle Modelle über die Modelleigenschaften verändert werden. Der in diesem Feld eingetragene Name stellt lediglich den Vorgabewert dar.

Vorgabe-Autoren-E-Mail-Adresse für neue Modelle

Die in diesem Eingabefeld eingetragene E-Mail-Adresse wird als Autoren-E-Mail-Adresse beim Anlegen von neuen Modellen verwendet. Der die Autoren-E-Mail-Adresse kann für individuelle Modelle über die Modelleigenschaften verändert werden. Die in diesem Feld eingetragene E-Mail-Adresse stellt lediglich den Vorgabewert dar.

Vorgabedateiformate beim Speichern

In diesen Auswahlfeldern kann festgelegt werden, welches Dateiformat jeweils in den "Speichern unter"-Dialogen für die jeweiligen Datenarten initial ausgewählt angezeigt werden soll.

(Vorgabewerte: "xml" für Modelle und Statistiken und "zip-komprimiertes xml" für Parameterreihen und Optimierereinstellungen.)

Sicherheitskopien von Modelldateien anlegen

Wenn diese Option aktiviert ist, wird beim Speichern einer Modelldatei ohne Eingabe eines neuen Dateinamens eine Sicherheitskopie-Datei der vorherigen Datei mit der Dateinamensendung bak angelegt.

(Diese Option ist als Vorgabewert deaktiviert.)

Benutzerdefinierter Zeilenbezeichner

Beim DDE-Zugriff auf eine Excel-Tabelle verwendet der Simulator einen Bezeichner für die Auswahl der Zeile. Für deutsche Excel-Versionen ist dies ein "Z" (wie Zeile) und für englische Versionen ein "R" (wie row). Im Normalfall verwendet der Simulator die deutsche oder die englische Version in Abhängigkeit von der gewählten Programmsprache. Weiche Excel-Programmsprache und Simulator-Programmsprache von einander ab, so kann es erforderlich sein, hier einen expliziten Wert für den Zeilenbezeichner anzugeben.

(Vorgabewert für diese Option ist ein leeres Feld.)

Benutzerdefinierter Spaltenbezeichner

Beim DDE-Zugriff auf eine Excel-Tabelle verwendet der Simulator einen Bezeichner für die Auswahl der Spalte. Für deutsche Excel-Versionen ist dies ein "S" (wie Spalte) und für englische Versionen ein "C" (wie column). Im Normalfall verwendet der Simulator die deutsche oder die englische Version in Abhängigkeit von der gewählten Programmsprache. Weiche Excel-Programmsprache und Simulator-Programmsprache von einander ab, so kann es erforderlich sein, hier einen expliziten Wert für den Spaltenbezeichner anzugeben.

(Vorgabewert für diese Option ist ein leeres Feld.)

Auflösung beim Speichern von Bildern

In diesem Feld kann angegeben werden, welche Auflösung in Pixeln sowohl in horizontaler als auch in vertikaler Richtung beim Speichern von Grafiken aus den Statistikergebnissen als Pixelgrafiken verwendet werden soll.

(Vorgabewert für diese Option: 2000 x 2000 Pixel.)

Bilder während der Animation speichern

Werden Screenshots über die entsprechende Symbolleistenschaltfläche direkt während der Animation eines Modells gespeichert, so werden diese ohne weitere Rückfrage in das hier angegebene Verzeichnis geschrieben. Auf diese Weise wird die Unterbrechung der Animation so kurz wie möglich gehalten.

(Vorgabewert für diese Option ist die Verwendung des Nutzerverzeichnisses.)

Simulationszeit in aufgezeichneter Animation einblenden

Wenn diese Funktion aktiviert ist, wird unten links in die als Video aufgezeichneten Animationen die aktuelle Simulationszeit eingeblendet. Während der Ausführung einer Animation im Simulator wird die Simulationszeit in der Statusleiste angezeigt. Die Statusleiste ist jedoch nicht Bestandteil des aufgezeichneten Videos.

(Diese Option ist als Vorgabewert aktiviert.)

  Sicherheit

Modelle mit potentiell kritische Stationen

Über dieses Auswahlfeld kann eingestellt werden, wie sich das Programm verhalten soll, wenn ein Modell geladen werden soll, welche Elemente enthält, die evtl. Auswirkungen auf das System haben können (Stationen, die Skriptausführung beinhalten oder die Daten in Dateien oder Datenbanken ausgeben). Es kann gewählt werden, das Laden immer zu erlauben oder immer zu verbieten oder aber den Nutzer zu fragen.

(Vorgabewert für diese Option: "Nachfragen".)

Einschränkungen anwenden auf

In diesem Auswahlfeld kann festgelegt werden, ob die Einstellungen in "Modelle mit potentiell kritische Stationen" für alle zu ladenden Modelle gelten sollen oder nur für Modelldateien, die aus dem Internet heruntergeladen wurden.

(Vorgabewert für diese Option: "Modelldateien aus dem Internet".)

Modelle, die Skripte enthalten, signieren

Ist diese Option aktiviert, so werden Modelle, die kritische Stationen wie z.B. Skripte enthalten, beim Speichern signiert, so dass beim Laden dieser Modelle auf demselben Rechner keine Sicherheitsfrage gestellt werden muss.

(Diese Option ist als Vorgabewert aktiviert.)

Ausführung externer Programme durch Skripte zulassen

Wenn diese Option aktiviert wird, können von Skripten aus externe Programme aufgerufen werden und der jeweilige Rückgabewert im Skript weiterverwendet werden. Wenn Skripte externe Programme aufrufen können, sind damit prinzipiell beliebige Veränderungen am Computer möglich. Damit stellt das Öffnen und Ausführungen von Modellen ein Sicherheitsrisiko dar. Es wird daher empfohlen, diese Option nur zu aktivieren, wenn dies unbedingt notwendig ist.

(Diese Option ist als Vorgabewert deaktiviert.)

Beim Anklicken von Links

Wird ein Link im Simulator angeklickt, so wird dieser nach einer Rückfrage an den Standard-Browser übergeben. Über diese Einstellung kann der Aufruf eines Browser gänzlich verboten werden.

(Vorgabewert für diese Option: "Nachfragen".)

  Update

Java-Version prüfen und warnen, wenn veraltet und unsicher

Ist diese Option aktiviert, so prüft der Simulator beim Programmstart, ob die aktuell installierte und verwendete Java-Version zum Zeitpunkt der Auslieferung des Simulators noch aktuell war und gibt ggf. eine Sicherheitswarnung aus.

(Diese Option ist als Vorgabewert aktiviert.)

Programm automatisch aktualisieren

Wenn diese Option aktiviert ist, prüft der Simulator in regelmäßigen Abständen, ob eine neue Version veröffentlicht wurde und installiert diese automatisch. Voraussetzung hierfür ist die Verwendung der Windows-Version des Simulators.

(Sind automatische Updates möglich, so sind diese per Vorgabe aktiv. Andernfalls werden per Vorgabe Updates gesucht und es wird auf diese hingewiesen.)
