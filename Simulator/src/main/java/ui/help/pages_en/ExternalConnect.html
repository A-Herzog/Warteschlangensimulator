Java plugins folder dialog

In Java code used at script stations can call Java code in external class files. Two techniques are available for this purpose:

Plugin interface

External classes can implement methods with the following signature:

public Object methodName(final scripting.java.RuntimeInterface runtime, final scripting.java.SystemInterface system, final Object data)

These methods can be accessed in the internal Java code via the following method:

sim.getSystem().runPlugin("className","methodName",data)

If a directory is selected in the dialog, the class files found in each case and the matching methods contained therein are automatically listed in the list view. Only those classes are listed which contain methods according to the signature described above.

Import of external classes

External classes located in the subdirectories of the specified directory can be made available directly via import statements. Note that the classes must not be located in the specified plugin directory itself, but in its subdirectories, and that a suitable package identifier has to be specified in the classes.

The import is done automatically by the simulator. No manual import configurations have to be made. Only the base directory where the external classes are located has to be specified.

General information

  • A general documentation of the available Java commands is available on the help page Documentation of Java commands.
  • If a class file was loaded or used by the simulator for the first time, it can remain locked until the simulator is terminated. This means that in particular it is not ensured that class files in the plugins directory can be exchanged at runtime of the simulator.
