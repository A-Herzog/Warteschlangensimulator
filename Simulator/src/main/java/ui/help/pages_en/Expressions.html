Calculation commands

In the Edit expression dialog you can define expressions to be calculated. The following calculation commands for mathematical expressions are available. If the input field is a field for an expression that is to be calculated to true or false, comparative operations are available in the "Comparisons" section. Otherwise (for example, for the calculation of the score of a client), only the rake commands, which at the end deliver a number, are available.

Variables

When calculating values in the context of a concrete client, the variables w (for the previous waiting time of the client), t (for the previous transfer time of the client) and p (for the clients's previous operating time) are always available. If the calculation is done for getting a clients score, the variable w does not contain the total waiting time of the current client but the waiting time of the current client at the current station.

Furthermore, all variables that are defined by an assignment element are always available. Before the first assignment of a value to a variable, it has the value 0.

Basic arithmetic operations

Supported instructions for the basic arithmetic operations: +, -, *, / and ^.

Additionally an If function is available. This functions expects an odd number of parameters:

If(condition1;value1;condition2;value2;...;valueElse)

If condition1>0 the function returns value1.
Otherwise condition2>0 is tested and if its fulfilled value2 is returned and so on.
If non of the conditions is fulfilled the function returns valueElse.

ErlangC(lambda;mu;nu;c;K;-1)Calculates the average queue length E[NQ]
ErlangC(lambda;mu;nu;c;K;-2)Calculates the average number of clients in the system E[N]
ErlangC(lambda;mu;nu;c;K;-3)Calculates the average waiting time E[W]
ErlangC(lambda;mu;nu;c;K;-4)Calculates the average residence time E[V]
ErlangC(lambda;mu;nu;c;K;-5)Calculates the average accessibility 1-P(A)
ErlangC(lambda;mu;nu;c;K;t)Calculates the the probability for the service level at the t seconds threshold P(W≤t)
