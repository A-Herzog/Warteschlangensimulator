Erlang B

The Erlang-B formula assumes that there is no waiting room in the model, so that clients are rejected to enter the system if they cannot be served directly.

Since the Erlang-B formula originally comes from the telecommunications sector, we are not talking about the number of operators here, but about the number of lines N. The quantity a indicates the work load - in this context also called offered work load. This value, the quotient of the arrival rate and the service rate, indicates how many operators or lines are required in any case in order to be able to serve the arriving clients in the long term.

The displayed value P(B) indicates the probability of an incoming client to encounter the system in a state in which there are already N clients in the system (i.e. all lines are busy). In this case, the new client is immediately rejected. When dimensioning a communications system in practice, the aim is to keep P(B) as small as possible.
