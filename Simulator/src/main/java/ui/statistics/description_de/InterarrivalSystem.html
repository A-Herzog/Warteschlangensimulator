Der unter Ankünfte laut Modell angegebene Wert entspricht der in den Modelleigenschaften angegebenen Anzahl an zu simulierenden Kunden. Der Wert Ankünfte laut Simulation kann jedoch leicht davon abweichen. Hintergrund ist, auf welche Art die Simulation beendet wird: Sofern nicht anders konfiguriert werden alle Ankünfte gezählt. Ist die Soll-Anzahl erreicht, so werden weiterhin Ankünfte generiert, diese Kunden jedoch als "letzter Kunde" markiert. Sobald einer dieser "letzten Kunden" einen Ausgang erreicht hat, wird die Simulation beendet. Bis dahin werden in den meisten Fällen bereits einige weitere, über die eingestellte Anzahl hinausgehende Kunden generiert worden sein.

Zwischenankunftszeiten

Eine Zwischenankunftszeit ist die Zeitspanne zwischen dem Eintreffen von zwei unmittelbar aufeinander folgenden Kunden am System. Es werden hier Mittelwerte, Standardabweichungen, Varianzen, Variationskoeffizienten, Schiefe, Exzess (Wölbung) und Quantile ausgegeben.

Die mittlere Zwischenankunftszeit wird mit E[I] bezeichnet (und z.B. in Sekunden oder Minuten gemessen). Der Kehrwert von E[I] ist die Ankunftsrate. Sie wird mit lambda (λ) (in in z.B. 1/Sekunde oder 1/Minute gemessen). Kleine Werte von E[I] bzw. große Werte von lambda bedeuten, dass innerhalb kurzer Zeit viele Kunden eintreffen.

Konfidenzintervalle

Zu den auf dieser Seite angezeigten Größen können optional Konfidenzintervalle ausgegeben werden. Zur Berechnung von Konfidenzintervallen müssen die einzelnen Messwerte stochastisch unabhängig sein, was bei den in einem Warteschlangensystem erfassten Größen üblicherweise nicht der Fall ist. Um dennoch Konfidenzintervalle bestimmen zu können, wird daher eine feste Anzahl an Werten zu einem Batch zusammengefasst. Die Batch-Größen werden dabei so gewählt, dass zwischen diesen (fast) keine Abhängigkeiten mehr bestehen. (Diese virtuelle Zusammenfassung von Messwerten hat nichts mit Ankunfts- oder Bedien-Batchen zu tun.) Die Mittelwerte pro Batch werden dann als Eingangsgröße für die Berechnung der Konfidenzintervalle verwendet.

Die Erfassung von Batch-Means-basierenden Konfidenzintervallen kann auf der Dialogseite Ausgabeanalyse im Modelleigenschaften-Dialog aktiviert werden. Dafür muss eine Batch-Größe vorgegeben werden. Diese kann vorab über den Menüpunkt Statistik-Batch-Größe bestimmen im Simulation-Menü automatisch bestimmt werden.
