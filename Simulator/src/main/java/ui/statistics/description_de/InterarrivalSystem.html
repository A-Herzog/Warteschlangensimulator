Der unter Ankünfte laut Modell angegebene Wert entspricht der in den Modelleigenschaften angegebenen Anzahl an zu simulierenden Kunden. Der Wert Ankünfte laut Simulation kann jedoch leicht davon abweichen. Hintergrund ist, auf welche Art die Simulation beendet wird: Sofern nicht anders konfiguriert werden alle Ankünfte gezählt. Ist die Soll-Anzahl erreicht, so werden weiterhin Ankünfte generiert, diese Kunden jedoch als "letzter Kunde" markiert. Sobald einer dieser "letzten Kunden" einen Ausgang erreicht hat, wird die Simulation beendet. Bis dahin werden in den meisten Fällen bereits einige weitere, über die eingestellte Anzahl hinausgehende Kunden generiert worden sein.

Zwischenankunftszeiten

Eine Zwischenankunftszeit ist die Zeitspanne zwischen dem Eintreffen von zwei unmittelbar aufeinander folgenden Kunden am System. Es werden hier Mittelwerte, Standardabweichungen, Varianzen, Variationskoeffizienten, Schiefe und Quantile ausgegeben.

Die mittlere Zwischenankunftszeit wird mit E[I] bezeichnet (und z.B. in Sekunden oder Minuten gemessen). Der Kehrwert von E[I] ist die Ankunftsrate. Sie wird mit lambda (λ) (in in z.B. 1/Sekunde oder 1/Minute gemessen). Kleine Werte von E[I] bzw. große Werte von lambda bedeuten, dass innerhalb kurzer Zeit viele Kunden eintreffen.
