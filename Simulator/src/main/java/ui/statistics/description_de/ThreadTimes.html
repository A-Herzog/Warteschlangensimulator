Wenn die Simulation auf mehrere CPU-Kerne aufgeteilt wird, so simuliert jeder Thread nicht eine feste Anzahl an Ankünften, sondern die Ankünfte werden gemäß des jeweiligen Simulationsfortschritts des jeweiligen Threads dynamisch zugewiesen. Dadurch kann die verfügbare CPU-Leistung optimal ausgenutzt werden. Im Idealfall führt diese dynamische Lastanpassung dazu, dass die Laufzeiten aller Threads identisch sind. Auf dieser Seite wird angezeigt, wie weit die Laufzeiten noch von dieser Idealvorstellung abweichen.
