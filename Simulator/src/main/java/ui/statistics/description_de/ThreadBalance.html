Wenn die Simulation auf mehrere CPU-Kerne aufgeteilt wird, so simuliert jeder Thread nicht eine feste Anzahl an Ankünften, sondern die Ankünfte werden gemäß des jeweiligen Simulationsfortschritts des jeweiligen Threads dynamisch zugewiesen. Dadurch kann die verfügbare CPU-Leistung optimal ausgenutzt werden. Auf dieser Seite wird angezeigt, wie viele Ankünfte welcher Thread verarbeitet hat.
