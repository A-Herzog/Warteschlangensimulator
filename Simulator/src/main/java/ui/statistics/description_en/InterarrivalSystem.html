The value specified by Arrivals according to model corresponds to the number of clients to be simulated as specified in the model properties. However, the value Arrivals according to simulation can deviate slightly. The background is the way in which the simulation is ended: Unless otherwise configured, all arrivals are counted. If the target number is reached, the system continues to generate arrivals, but marks these clients as "last client". As soon as one of these "last clients" has reached an exit, the simulation is terminated. By then, in most cases some more clients beyond the defined number will have been generated.

Inter-arrival times

An inter-arrival time is the time span between the arrival of two immediately consecutive clients at the system. Average value, standard deviation, variances, coefficients of variation, skewness and quantiles are displayed.

The average inter-arrival time is designated by E[I] (and is measured in seconds or minutes, for example). The reciprocal of E[I] is the arrival rate. It is designated by lambda (λ) (measured in e.g. 1/second or 1/minute). Small values of E[I] or large values of lambda mean that many clients arrive within a short time.
