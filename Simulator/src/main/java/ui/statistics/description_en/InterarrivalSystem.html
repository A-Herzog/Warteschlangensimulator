The value specified by Arrivals according to model corresponds to the number of clients to be simulated as specified in the model properties. However, the value Arrivals according to simulation can deviate slightly. The background is the way in which the simulation is ended: Unless otherwise configured, all arrivals are counted. If the target number is reached, the system continues to generate arrivals, but marks these clients as "last client". As soon as one of these "last clients" has reached an exit, the simulation is terminated. By then, in most cases some more clients beyond the defined number will have been generated.

Inter-arrival times

An inter-arrival time is the time span between the arrival of two immediately consecutive clients at the system. Average value, standard deviation, variances, coefficients of variation, skewness, excess kurtosis and quantiles are displayed.

The average inter-arrival time is designated by E[I] (and is measured in seconds or minutes, for example). The reciprocal of E[I] is the arrival rate. It is designated by lambda (λ) (measured in e.g. 1/second or 1/minute). Small values of E[I] or large values of lambda mean that many clients arrive within a short time.

Confidence intervals

For the performance indicators displayed on this page, confidence intervals can optionally be output. To calculate confidence intervals, the individual measured values have to be stochastically independent, which is usually not the case with the indicators recorded in a queueing system. In order to be able to determine confidence intervals in spite of this, a fixed number of values is therefore combined to form a batch. The batch sizes are chosen in such a way that (almost) no dependencies exist between them. (This virtual grouping of measured values has nothing to do with arrival or service batches). The mean values per batch are then used as input variable for the calculation of the confidence intervals.

The recording of batch-means-based confidence intervals can be activated on the Output analysis page in the Model properties dialog. A batch size has to be specified for this. This can be determined automatically in advance via the menu item Determine statistics batch size in the Simulation menu.
